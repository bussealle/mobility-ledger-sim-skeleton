# 04. ステークホルダモデル

## 50主体・6レイヤ構造

50のステークホルダは、行動プロファイル(役割6次元 + 状態関与8次元)から階層クラスタリングで6レイヤに整理されている。`data/stakeholders.json` の `layer` フィールドを参照する。

## 役割の機能軸(6分類)

ステークホルダがイベントに関与する役割は以下の6つ:

| 役割 | 意味 | 典型例 |
|------|------|--------|
| 起点(originator) | イベントを発生させる主体・意思決定の出発点 | EV007ファイナンス申込の顧客 |
| 対象(subject) | イベントの対象・主体的な状態変化を受ける主体 | EV012新規登録の車両所有者 |
| 実行(executor) | イベントを実際に操作・処理する主体 | EV019定期メンテナンスの整備工場 |
| 確認(attestor) | 事実を確認・記録・検証する第三者 | EV030車検の検査機関 |
| 受益(beneficiary) | 経済的・実質的便益を受ける主体 | EV011頭金決済のキャプティブ |
| 責任(liability) | リスク・責務・債務を負う主体 | EV031月次払いの借り手 |

同じステークホルダが、イベントによって異なる役割を演じることもある(キャプティブが起点だったり受益だったり)。**役割はイベント文脈固有**である。

## STK02 キャプティブの位置づけ

キャプティブファイナンス(STK02)はL6に属する。データ上の特徴は:

- 関与イベント数: 40件(全128件中)
- 役割分布:
  - 起点: 15件(最多。価格・契約・残価・回収のドライバー)
  - 実行: 11件
  - 受益: 10件
  - 対象: 3件
  - 責任: 1件

副目的(γ)では、プレイヤーがSTK02役を兼ねるオプションを提供する。

## レイヤ別の特性とキャプティブの関係

各レイヤとキャプティブの距離・関係は以下の通り:

### L6(同居)
キャプティブと同じレイヤ。**最終的な損益・責任・資金の流れの当事者**。OEM・顧客・投資家・保険会社など。キャプティブにとって**直接の取引先**。

### L1(隣接・代替可能)
キャプティブの執行能力の代替・補完。SPV・サービサー・銀行・配車PF・残価保証提供者など。キャプティブが**自前でやるか、外部化するか**の選択肢を持つ領域。

### L2(対峙・連携)
車両に物理的に触るサービス層。ディーラー・整備工場・テレマティクス・OTA配信。キャプティブにとって**データの源泉**であり、**顧客接点の重要な担い手**。

### L4(対峙)
中古市場・フリート運営・物流・海外販社。キャプティブにとって**残価が決まる場所**。

### L5(必須インフラ)
車検・登録・査定・税関。回避不可能な認証層。キャプティブのトランザクションの**ゲート**になる。

### L3(終末)
事故修理・解体・サルベージ。キャプティブにとって**回収・損失処理**の領域。

## ステークホルダの可視化(Phase 3)

Phase 3 で実装する「ステークホルダ星座ビュー」は、50主体を6レイヤ別に画面上に配置する。レイアウトのヒント:

```
        L5 (上部・中央)        ← 認証・規制
   L4左          L4右          ← 物理占有・流通
        L6 (中央・車両近傍)    ← 主導・受益
   L1左          L1右          ← 金融実行
        L2 (中央下部)          ← 車両サービス
        L3 (最下部・暗色)      ← 終末処理
```

中央に車両アイコンを配置し、イベント発火時に関与する主体だけが点灯し、車両との間に**役割色の線**が伸びる。

## ステークホルダ統計の出力(Phase 4)

プレイ後の統計として、各STKの:

- 登場回数
- 役割分布(起点・実行・確認・受益・責任)
- 累積金銭フロー(ある場合)
- 関与の濃度(イベント数 × 平均関与時間)

を出力する。これがプレイ後の議論の素材になる。

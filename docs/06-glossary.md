# 06. 用語集

## ドメイン用語(車両ライフサイクル)

| 用語 | 意味 |
|------|------|
| ライフサイクル | 車両の製造から廃車・素材化までの一連の期間。本プロジェクトでは15年が標準想定 |
| 相 (Phase) | ライフサイクル上の大きな段階。10値(Production, Distribution, ..., EOL) |
| 残価 (Residual Value) | リース・残価設定型ローンで契約時に確定される、契約満了時の想定価値 |
| 残価保証 | 残価が市場価格を下回った場合に差額を補填する保証商品 |
| バルーン | 残価設定型ローンの満了時に発生する一括清算金 |
| アテステーション | 第三者による事実確認の宣言・記録(SoH宣言、走行距離認証読取等) |
| SoH (State of Health) | 主にEVバッテリーの健全性指標。0.0〜1.0で表現 |
| 全損 | 経済的修理不能の状態。保険会社が宣言する |
| サルベージ | 全損宣言後の車両の市場流通(廃車部品取り、輸出等) |
| フロアプラン | ディーラー在庫車両への金融供給(在庫融資) |
| 証券化 (Securitization) | オートローン債権をプール化してABSとして投資家に販売する金融取引 |
| ABS (Asset-Backed Securities) | 資産担保証券。証券化の最終商品 |
| SPV (Special Purpose Vehicle) | 証券化のためにアセットを保有する導管会社 |
| サービサー | ローン債権の回収業務を執行する代行会社 |
| 越境 | 国境を越えた車両の移動。輸出・輸入・第三国再輸出等 |

## キャプティブ・ファイナンスの用語

| 用語 | 意味 |
|------|------|
| キャプティブ・ファイナンス | OEM系列の自動車金融会社。販売金融とリースを主業務とする |
| 信用製造 (Credit Manufacturing) | キャプティブの本質を「信用を生産する装置」と捉える視座 |
| 介入機会 | キャプティブが新たに参入できる、もしくは支配力を強化できるトランザクション領域 |
| MON (Mobility Orchestration Network) | モビリティアセットのオーケストレーションネットワーク |
| MOA (Mobility Oriented Account) | 個別車両を主体として持つアカウント概念 |

## システム設計用語

| 用語 | 意味 |
|------|------|
| ステートマシン | 有限の状態とその間の遷移で挙動を表現するモデル |
| 純粋関数 | 同じ入力に対し同じ出力を返し、副作用を持たない関数 |
| 不可逆イベント | 一度発火すると以前の状態に戻れないイベント(廃車、永久抹消等) |
| 連鎖イベント (cascade) | 別イベントの結果として自動発火するイベント |
| 確率発火イベント (probabilistic) | 時間経過に従って確率的に発火するイベント |
| 周期発火イベント (time) | 一定周期で必ず発火するイベント(税、車検、月次払い等) |
| 選択イベント (choice) | プレイヤーが能動的に選んで発火させるイベント |
| 事前条件 (precondition) | イベント発火可能性を判定する状態条件 |
| 事後効果 (postcondition) | イベント発火後の状態変更の指定 |
| Ledger | トランザクションの累積ログを保持するデータ構造 |
| シナリオ | 市場・規制・確率パラメータの初期値セット |
| シード(乱数) | 乱数生成器の初期値。同じシードからは同じ乱数列が生成される |

## 役割6分類(イベント関与の機能軸)

| 役割 | 英語 | 意味 |
|------|------|------|
| 起点 | originator | イベントを発生させる主体 |
| 対象 | subject | イベントの対象・状態変化を受ける主体 |
| 実行 | executor | イベントを実際に操作・処理する主体 |
| 確認 | attestor | 事実を確認・記録・検証する第三者 |
| 受益 | beneficiary | 経済的・実質的便益を受ける主体 |
| 責任 | liability | リスク・責務・債務を負う主体 |

## 状態変化軸(7+1)

| 軸 | 英語 | 意味 |
|----|------|------|
| 所有 | ownership | 法的所有権の移転 |
| 占有 | possession | 物理的占有・場所の変化 |
| 状態 | condition | 物理状態・健全性の変化 |
| 価値 | value | 経済価値の再評価 |
| 規制 | regulatory | 登録状態・適合状態の変化 |
| 債務 | financial | 債権債務関係の変化 |
| データ | data | 履歴・属性情報の更新 |
| 不可逆 | irreversible | このイベントが原則不可逆かどうかのフラグ |

## レイヤ(クラスタリングで抽出された6層)

| ID | 名称 | 主体数 | 性格 |
|----|------|-------|------|
| L1 | 金融サービス実行層 | 9 | 他者の意思を金融・契約として執行 |
| L2 | 車両サービス・データ提供層 | 10 | 物理車両への作業とデータ更新 |
| L3 | 終末処理層 | 5 | 不可逆処理の専門家 |
| L4 | 物理占有・流通実行層 | 11 | 車両を実際に占有・運用 |
| L5 | 認証・登録インフラ層 | 6 | 事実を認証する第三者 |
| L6 | 主導・受益・責任層 | 9 | 損益・責任の最終帰属先 |

# Phase 3: 体験レイヤ(星座 / オーラ / 演出)

## 目的

Phase 2 で「動くMVP」が出来た上に、**プレイヤーに「実感」を生む視覚言語**を載せる。具体的には3つの可視化レイヤを重ねて、車両ライフサイクルを「体感する装置」に変える。

## 着手の前提

- Phase 2 の受入基準を全て満たしていること
- `npm run dev` で起動でき、最初から最後までプレイできること
- core/ のテストが通っていること

これらが満たされていない状態で Phase 3 に進まないこと。

## このPhaseで実現すべき体験

`docs/01-design-principles.md` の体験デザイン原則を再読すること。Phase 3 は特に以下の原則を実装する:

- 原則1: 状態が見えること(7属性の視覚化を強化)
- 原則2: 関与が可視化されること(ステークホルダ星座 + 役割線)
- 原則5: 選択の重みを感じること(不可逆イベントの演出)

## スコープ

### 1. ステークホルダ星座ビュー(StakeholderConstellation.tsx)

50主体を6レイヤ別に画面上に配置する。中央に車両、周囲に主体を配置。

レイアウト方針(`docs/04-stakeholder-model.md` 参照):

```
        L5 (上部)        ← 認証
   L4左          L4右    ← 物理占有
        L6 (中央)        ← 主導受益
   L1左          L1右    ← 金融実行
        L2 (下部)        ← 車両サービス
        L3 (最下部)      ← 終末処理
```

- 各レイヤを色分け(`docs/04-stakeholder-model.md` のレイヤ色)
- 主体はラベル付きの円。普段は薄表示
- イベント発火時:関与する主体だけ点灯(レイヤ色に明るくなる)し、車両との間に**役割色**(起点=黄、対象=薄青、実行=薄緑、確認=薄ピンク、受益=淡黄、責任=淡橙)の線が引かれる
- 線は1〜2秒のフェード演出で表示し、消える
- イベント発火が累積すると、頻繁に登場した主体ほど少し大きく/濃く描画される(累積関与の可視化)

### 2. 車両オーラ(VehicleAura.tsx)

中央の車両アイコンに、7属性をオーラとして表示:

- **残債オーラ**: 赤色、強さは未払残高 / MSRP に比例
- **SoH オーラ**: 青色、薄さは(1 - soh)に比例。BEVのみ
- **走行距離オーラ**: 灰色、強さは累積走行距離に比例
- **データ蓄積オーラ**: 緑色、強さは history_score に比例
- **規制リスクオーラ**: 橙色、未払い違反金 / 期限切れ近接 で点滅

オーラは車両アイコンを中心とした同心円や放射状のグラデーションで表現する。CSSのbox-shadow多重がけ、SVGのfilter、またはCanvas2Dで実装可能。

複数のオーラを重ねて表示するため、**色の重ね合わせと透明度設計**を慎重に行う。視覚的に整理されていないと「ぐちゃぐちゃ」になる。

### 3. トランザクション展開パネル(TransactionPanel強化)

Phase 2 ではテキスト主体だったパネルを、以下の演出で拡張:

- イベント発火の瞬間にカード型でポップ表示(右下からスライドイン、3秒後フェードアウト)
- カード内に:
  - イベント名
  - カテゴリバッジ
  - 関与STKのアイコン(円表示)を役割色付きで横並び
  - 状態変化フラグ(変化した属性を Y のフラグでバッジ表示)
- カード本体クリックで詳細展開(展開時はフェードアウトしない)

### 4. 不可逆イベントの演出

`state_changes.不可逆 = true` のイベントを発火しようとしたとき:

- 確認ダイアログ「この選択は戻れません。続行しますか?」を表示
- 確認後、画面に短いフラッシュ演出(0.3秒の白オーバーレイ)
- TransactionPanel のカードに「⚠ 不可逆」バッジを赤色で表示

これは原則5の実装。

### 5. ライフラインビュー強化(Timeline)

Phase 2 のテキストベースのリストから、横軸=時間の帯状ビューへ:

- 上から下にカテゴリ別の帯(製造/流通/小売/金融/使用/...)
- 各帯にイベントが時系列でプロットされる(色付きドット)
- ホバーで詳細表示
- 横スクロールで15年全期間を見渡せる
- 現在時刻のカーソル線

### 6. 時間進行のテンポ調整

Phase 2 では「N日進める」ボタンだけだったが、Phase 3 では:

- **平穏期スキップ**: 確率/周期イベントが何も発火しない期間は自動的にスキップ(visual fast-forward)
- **活発期スロー**: イベント密集期はカードを順番に表示しながら進む
- プレイヤーは「次のイベントまで進む」「年末まで進む」「契約満了まで進む」を選択可能

## 非ゴール(Phase 3 では実装しない)

- シナリオ選択UI → Phase 4
- 複数台同時シミュレーション → Phase 4
- 統計ビュー(プレイ後の集計画面) → Phase 4
- 共有可能URL → Phase 4
- BGM・効果音 → 全Phase非対象
- 3D表現・WebGL → 全Phase非対象。SVGとCanvas2Dで完結する

## 受入基準

1. ステークホルダ星座ビューが画面に表示され、50主体すべてが視認できる
2. イベント発火時に、関与する主体だけが点灯し、役割色の線が伸びる(2秒程度のフェード演出)
3. 車両アイコンにオーラが表示され、状態変化に応じて色・強度が変わる
4. トランザクションカードがイベント発火時にポップ表示され、関与STKと状態変化が一目でわかる
5. 不可逆イベントは確認ダイアログを経て発火し、視覚的なフラッシュ演出がある
6. ライフラインビューが時系列で横展開され、ホバーで詳細が見える
7. 時間進行が「自動スキップ」と「展開表示」を使い分けている
8. パフォーマンス: 50主体・100イベント発火後でも30fpsを保つ
9. 既存のPhase 2機能(プレイ完走、テスト全通過)が壊れていない

## 推奨実装順序

### Step 1: 不可逆イベントの確認ダイアログ

最も簡単で効果が大きい。Phase 2 のEventPickerに追加するだけ。

### Step 2: トランザクションカードの強化

役割色付きSTKアイコンの横並び表示。CSS Animationでスライドイン。

### Step 3: ステークホルダ星座ビュー(静的)

レイアウト・配置・主体の表示。最初はインタラクションなしで構わない。

### Step 4: ステークホルダ星座ビュー(動的)

イベント発火と連動した点灯・線描画。

### Step 5: 車両オーラ

最初は1属性(残債)だけで実装し、見た目を確認してから他属性に展開。

### Step 6: ライフラインビュー

横軸時間・縦軸カテゴリの帯ビュー。データはLedgerから直接構築。

### Step 7: 時間進行のテンポ調整

UI側でループを回すだけなので最後で良い。

### Step 8: パフォーマンス調整

主に星座ビューでの再描画削減(useMemo, 線をSVGとして1回だけ描く等)。

## 設計上の注意

### 演出は控えめに

派手なアニメーション・パーティクル・ネオン光は使わない。**情報を伝える最小限の演出**に留める。`docs/05-architecture.md` のスタイル方針を継承(白背景・薄い境界線・フラット)。

### Component分割

星座ビューは大きくなるので、以下に分割:

- `StakeholderConstellation.tsx`(本体・配置)
- `StakeholderNode.tsx`(個別主体)
- `RoleLine.tsx`(役割線)
- `LayerLabel.tsx`(レイヤ名表示)

各Componentは表示専用で、イベント連動は親から props で受け取る。

### Logic不変の原則を守る

Phase 3 で core/ には**ほぼ手を入れない**。例外:

- ledger に「直近イベントのハイライト時刻」を記録する程度の補助情報を追加
- selectors.ts に派生情報計算(関与頻度ランキング等)を追加

これ以上 core/ をいじりたくなったら、それは設計が間違っているサイン。

### data/ も基本不変

cascade_chains.json を新設するのは可。それ以外の data/*.json は Phase 2 のまま使う。

### CSS Animation vs Canvas

- ステークホルダ星座: SVG + CSS transition で十分
- 車両オーラ: SVG filter または CSS box-shadow で十分
- トランザクションカード: CSS animation で十分
- パーティクル等: 不要

Canvas2D が必要になるのは、50ノード+50線が同時に動く場合のみ。最初はSVG/CSSで実装し、パフォーマンス問題が出たら Canvas に置き換える。

### a11y

- aria-label を主要要素に設定
- キーボードでイベント選択ができる(Tab + Enter)
- 色覚多様性配慮: 役割の区別は色だけでなく線種(実線/点線/破線)も併用する
